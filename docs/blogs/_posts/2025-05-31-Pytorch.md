### Batch Normalization

Batch normalization is used to reduce the covariance when the weights get transferred from one layer to another layer in a neural network. It will increase the processing speed too.

- `torch.var`, back propagation
- Train — biased version: 1/n
- Test — unbiased version: 1/(n-1)

[Ioffe and Szegedy - 2015 - Batch Normalization Accelerating Deep Network Training](https://arxiv.org/abs/1502.03167)

### Backpropagation, `torch.nn` vs `torch.nn.functional`

The main difference between `nn.functional.xxx` and `nn.Xxx` in PyTorch is that the former lacks state while the latter maintains it. If you use the functional version for a linear layer, you'll need to manage the weights yourself (passing them to the optimizer or moving them to GPU). With `nn.Xxx`, it handles these tasks through `.parameters()` or `.to(device)`.

### Running on CUDA

- `to(device='cuda')`, `to(device='cpu')`, `cpu()`, `cuda()`
- NumPy is 64-bit float, PyTorch is 32-bit float

```python
p = torch.tensor([1, 2])
p_t = p.to(device='cuda')
# Perform any operation and return to CPU
p_cpu = p_t.to(device='cpu')

# Shorthand methods:
p_cpu = p_t.cpu()     # default CPU
p_cpu = p_t.cuda()    # default cuda is 0
p_cpu = p_t.cuda(1)   # cuda device 1
```

### Neural Networks

Mathematical expressions that take input data and weights/parameters in a neural network in forward pass. A loss function measures the accuracy of predictions. The loss will be low when predictions match targets. We manipulate the loss function to get low loss for good predictions. Then we backward the loss using back propagation to get gradients, tune parameters, and decrease the loss. Iterating this process many times is called gradient descent.

CNN, tanh, maxpooling, then take shape of pooling and view of it, linear output.
