[Dataform](https://app.notion.com/p/nsr6/Dataform-3f005b97e27e8090b2abd07db50848e6?source=copy_link) simplifies data processing architecture.

- Bought by Google in 2020
- Integrated into BigQuery
- Managing complex workflows
- Free service (pay for BigQuery)
- Dataform handles dependencies between tables

What is the source for this at the project level? Like we loaded the data from CSV files or DB, loaded it to BigQuery without transformation, then use Dataform.

## ETL vs ELT

| | ETL | ELT |
|---|---|---|
| Process | Extract → Transform → Load | Extract → Load → Transform |
| Stability | More stable and compliant data analysis | Faster and more flexible data analysis |
| Cost | Higher storage and compute costs | Lower cost and lower maintenance |
| Setup | Takes longer to set up because transformation logic must be built first, but data in warehouse is clean and stable from day one | This is the approach cloud warehouses made possible — storage is cheap, load everything and figure out transformations later |

> ELT is the dominant approach now, and it's the one we'll be working with. Dataform/dbt fits squarely into the "T" of ELT — it runs transformations inside the warehouse using SQL.
