# SEO Fixes for Search Engine Visibility

Date: 2026-06-22

## Problem

The site was not visible on search engines because:
1. Missing `jekyll-seo-tag` gem in Gemfile (listed in config but never installed)
2. No `robots.txt` file to guide crawlers
3. Missing meta descriptions on all pages
4. Generic site description in `_config.yml`

## Discussion

Analyzed `_config.yml`, `Gemfile`, `index.md`, and blog post front matter. Found that `jekyll-seo-tag` was listed as a plugin in `_config.yml` but absent from `Gemfile`, meaning it was never installed. Without it, no SEO meta tags (og:title, og:description, canonical URL, structured data) were generated. Also lacked `robots.txt` which search engines need to discover the sitemap.

Google Search Console verification file (`google5d3786c839de3815.html`) was already present.

## Code / Commands

```bash
# Add missing gem to Gemfile
echo 'gem "jekyll-seo-tag"' >> docs/Gemfile

# Install the gem
cd docs && bundle install

# Build to verify
bundle exec jekyll build

# Verify SEO tags in output
head -20 docs/_site/index.html
```

## Solution

1. Added `gem "jekyll-seo-tag"` to `docs/Gemfile`
2. Created `docs/robots.txt` pointing to sitemap.xml
3. Added keyword-rich `description:` to `_config.yml`, `index.md`, and all 27 blog posts
4. Ran `bundle install` and verified build succeeded with proper SEO tags
